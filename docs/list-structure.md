# List Structure

This document defines the structure of the SharePoint Requirements Register.

Each column is designed to support structured requirement tracking, stakeholder visibility, and iterative development.

---

## Core Fields

### Requirement ID
- Type: Single line of text
- Example: R-01, R-02
- Purpose: Unique identifier for each requirement

---

### Requirement Title
- Type: Single line of text
- Purpose: Short, descriptive name of the requirement

---

### Description
- Type: Multiple lines of text
- Purpose: Detailed explanation of the requirement

---

### Status
- Type: Choice
- Example values:
  - Draft
  - In Progress
  - Completed
- Purpose: Track current state of the requirement

---

### Iteration
- Type: Choice or text
- Example values:
  - I-01
  - I-02
- Purpose: Track development cycles

---

### Due Date
- Type: Date
- Purpose: Track deadlines and enable calendar view

---

### Priority
- Type: Choice
- Example values:
  - Low
  - Medium
  - High
- Purpose: Indicate importance of the requirement

---

### Owner
- Type: Person or text
- Purpose: Identify who is responsible for the requirement

---

### Stakeholders
- Type: Lookup (linked to Stakeholder Register)
- Purpose: Connect requirements to relevant stakeholders

---

### Related Process / BPMN Reference
- Type: Single line of text
- Purpose: Link requirements to BPMN models or related artifacts

---

### Requirement Type
- Type: Choice
- Example values:
  - Functional
  - Design
  - Analysis / Enabling
  - Meta / Enabling
- Purpose: Categorize requirements

---

### Notes
- Type: Multiple lines of text
- Purpose: Capture additional context, decisions, or references
